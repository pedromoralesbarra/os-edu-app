Introducción

Esta aplicación te guía paso a paso para entender cómo funciona una computadora y su sistema operativo. Cada sección explica componentes, su función y relaciones entre ellos.

Hardware de la computadora

1. Componentes principales

  • CPU: Unidad central de procesamiento; ejecuta instrucciones y controla otros componentes.
  • RAM: Memoria volátil usada por procesos en ejecución.
  • Almacenamiento: HDD, SSD, NVMe; retienen datos permanentemente.
  • Placa base (motherboard): Conecta componentes y ofrece buses de datos.
  • GPU: Procesamiento gráfico / aceleración (también usado para cómputo paralelo).
  • Periféricos: Teclado, ratón, cámaras, impresoras, etc.

2. Buses y comunicaciones

Los buses (PCIe, SATA, USB) conectan dispositivos y permiten transferencia de datos. El chipset y el controlador de memoria gestionan acceso y arbitraje.

Software y Sistema Operativo

El software se divide en aplicaciones (userland) y sistema (OS). El sistema operativo medía entre hardware y aplicaciones: gestiona procesos, memoria, I/O, seguridad y sistema de archivos.

Capas típicas

  1. Firmware (BIOS/UEFI)
  2. Bootloader
  3. Kernel
  4. Drivers
  5. Servicios del sistema
  6. Aplicaciones de usuario

El Kernel

El kernel es el núcleo del sistema operativo. Controla recursos, gestiona procesos, memoria, dispositivos y proporciona abstracciones (p. ej. archivos, sockets).

Tipos de kernel

  • Monolítico: Todo el núcleo (scheduler, filesystem, drivers) en una única imagen (ej. Linux tradicional).
  • Microkernel: Núcleo minimalista; servicios (drivers, filesystems) corren en espacio de usuario (ej. MINIX, QNX).
  • Híbrido: Compromiso entre ambos (ej. algunos diseños de Windows).

Servicios clave del kernel

Scheduler (planificación), gestión de memoria (virtual memory), manejo de interrupciones, llamadas al sistema (syscalls), comunicación entre procesos (IPC).

Procesos y concurrencia

Un proceso es una instancia de un programa en ejecución. Contiene su espacio de direcciones, hilos (threads), descriptores de archivos y estado.

Hilos

Los hilos permiten concurrencia dentro de un proceso. Pueden ser gestionados por el kernel (kernel threads) o por librerías (user threads).

Context switching

El scheduler cambia el contexto entre procesos/hilos guardando registros CPU y estado para reanudar luego.

Memoria y almacenamiento

Memoria virtual

El kernel presenta memoria virtual a procesos, trabajando con tablas de páginas y la MMU. Esto permite aislamiento y uso de más memoria que la física mediante swapping.

Gestión de memoria

Asignación de páginas, cachés, TLB, swapping/paging y protección de memoria son responsabilidades críticas.

Dispositivos y controladores (drivers)

Los drivers permiten al kernel comunicarse con hardware. Se encargan de inicializar dispositivos, manejar interrupciones y exponer interfaces a capas superiores.

Ejemplos: drivers de red, almacenamiento, gráficos y USB.

Proceso de arranque (Boot)

Secuencia típica: BIOS/UEFI → Bootloader (GRUB) → Kernel → Init/Systemd → Servicios y login.

El firmware inicializa hardware mínimo y transfiere control al bootloader que carga el kernel en memoria.

Sistemas de archivos

Los sistemas de archivos organizan datos en almacenamiento persistente. Ejemplos: FAT, NTFS, ext4, Btrfs. Proporcionan metadatos, journaling y permisos.

Redes y seguridad

El kernel incluye pila de red (TCP/IP) y mecanismos de seguridad: permisos, ACLs, SELinux/AppArmor, firewalls y cifrado.
